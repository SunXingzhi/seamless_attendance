## 服务器部署步骤

### 1. 前端构建
```bash
cd Fronted
npm install
npm run build
```

### 2. 后端构建
```bash
cd seamless_attendance
mvn clean package -DskipTests
```

### 3. 服务器部署

#### 3.1 复制文件
- 将前端`dist`目录复制到服务器`/path/to/frontend/dist`
- 将后端`target/seamless_attendance-1.0.0.jar`复制到服务器`/path/to/backend/`

#### 3.2 配置Nginx
编辑`/etc/nginx/nginx.conf`或创建新的配置文件：

```nginx
server {
    listen 80;
    server_name server_ip;

    # 前端静态资源
    location /seamless_attendance {
        alias /path/to/frontend/dist;
        index index.html;
        try_files $uri $uri/ /seamless_attendance/index.html;
    }

    # 后端API代理
    location /seamless_attendance/api {
        proxy_pass http://localhost:8080/seamless_attendance/api;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket代理
    location /seamless_attendance/ws {
        proxy_pass http://localhost:8080/seamless_attendance/ws;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

重启Nginx：
```bash
systemctl restart nginx
```

#### 3.3 启动后端服务
```bash
cd /path/to/backend
java -jar -Dspring.profiles.active=prod seamless_attendance-1.0.0.jar > backend.log 2>&1 &
```

### 4. 验证部署
- 访问：`http://server_ip/seamless_attendance`
- 检查API是否正常：`http://server_ip/seamless_attendance/api/devices/all`
- 检查WebSocket连接是否正常

## 注意事项
- 确保服务器上的MySQL数据库已创建并配置正确
- 确保Mqtt服务已运行
- 确保8080端口未被占用
- 确保Nginx配置正确，特别是路径映射

## 故障排查
- 检查Nginx错误日志：`/var/log/nginx/error.log`
- 检查后端服务日志：`/path/to/backend/backend.log`
- 检查前端控制台错误信息
- 确保所有路径配置一致
