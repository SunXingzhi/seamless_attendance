# Seamless Attendance Framework

每个工作室应该有一个管理员，还需要有一个总管理员账号（这个可以设计出来接口）。

## Fronted Demand

### 人员管理

每个人呈现块状展示。每行人员区块的数量根据浏览器的大小而改变（3/2/1变化）。每个区块需要显示的人员信息：

| 信息			 | 例子												 |
| :------------- | ---------------------------------------------------- |
| 姓名			 | 张三												 |
| 出勤情况(当前) | 缺勤/到勤/迟到										 |
| 学号/工号		| 22202432201XXXX										|
| 职称			 | XXX工作室员工/工作室管理员							 |
| 工作/任务		| 硬件设计/软件设计									|
| 联系方式		 | qq/wechat/phone number（需要选定一个唯一的联系方式） |

### 统计

统计菜单栏中支持查看近一周/一个月/半年/一年/**选定日期**的某个人/整个工作室的出勤次数、出勤率（出勤次数对于每年工作日的占比）。（选择栏）具体分成组件为：

| 时间		 | 人员			 | 统计内容				|
| ---------- | -------------- | --------------------- |
| week		 | 总体			 | 工作室活跃度/满勤次数 |
| month		| certain person | 个人活跃度/出勤次数	 |
| half year	| certain person | 个人活跃度			|
| year		 | certain person | 个人活跃度			|
| self-range | certain person | 个人活跃度			|
